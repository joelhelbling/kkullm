# Kkullm

## Choosing a name
> I want to come up with a name for a new agent orchestration project.  It will implement a classic blackboard pattern, with similarities to kanban and slack-like team chat.  So I'm thinking about the word _pull_, or perhaps the French, Spanish or Korean word for pull.  The sense in this word is that the system will revolve around agents pulling cards which are relevant/applicable to them and be prompted by the content and state of the card.

### Korean

- **당김 (Danggim)** — the noun form of *dangida* (당기다), meaning "pulling" or "a pull." Smooth to say, unusual in the Western dev namespace.
- **당 (Dang)** — the bare stem. Ultra-short. Has a satisfying snap to it. Could read as slightly irreverent in English though.
- **끌림 (Kkeullim)** — from *kkeulda* (끌다, to pull/drag/attract). Has a connotation of being *drawn toward* something, which maps well onto agents being attracted to relevant cards.

> I like Kkeullim.  Can we play with some alternative spellings to make it more catchy in English while remaining obviously derived from the Korean?

Fun exercise. Let's break down what makes 끌림 feel Korean so we know what to preserve:

- The **double-k** (ㄲ) — the tense/fortis consonant. This is the most distinctively Korean visual signature.
- The **"eu"** vowel (ㅡ) — has no English equivalent; romanizing it is always a creative choice.
- The **double-l** — artifact of ㄹ appearing at both end of first syllable and start of second.
- The **-im** ending — soft, nominal, ties it together.

---

### Spelling variants to consider

| Spelling | Notes |
|---|---|
| **Kkeullim** | Standard romanization. Correct but a bit clunky. |
| **Kkullim** | Swap *eu* → *u*. Punchier, easier for English readers. Still obviously Korean. |
| **Kkulim** | Drop one *l* too. Minimal, very clean. Reads almost like a brand already. |
| **Kkeulim** | Keep *eu*, drop one *l*. Splits the difference. |
| **Kkülim** | Umlaut nod to the ㅡ sound. Slightly precious but distinctive. |
| **Kkeul** | Truncate to just the "pull" morpheme. Tighter, but loses the nominal feel. |

---

### My shortlist

**Kkullim** — keeps the double-k Korean signature, the -lim ending, but swaps the awkward *eu* for *u*. Visually striking, short, utterly unique in the dev namespace.

**Kkulim** — one step further. Four consonants bookending two vowels. Has a real product-name economy to it. Easy to say: *"KOO-lim."*

**Kkeul** — if you want something even more minimal, just the verb root. Could work as a CLI tool name especially.

I'd lean **Kkullim** or **Kkulim** — the double-k does a lot of work signaling the origin without needing explanation.

### Decision

What do you think of **Kkullm**?

Interesting move — dropping the *i* to get a consonant cluster ending.

**What works:**
- Very terse and technical-feeling — fits well in a dev/CLI context
- The `llm` at the end is a happy accident: anyone in the AI space will instantly see **LLM** embedded in it, which is thematically perfect for an agent orchestration system
- Still visually Korean due to the double-k opener

**What's tricky:**
- Pronunciation becomes ambiguous — is it *"KULL-m"*? *"KULL-um"*? The missing vowel creates a stumble
- Loses the clean *-im* nominal ending that made 끌림 feel like a noun ("a pulling")

**The LLM easter egg is genuinely clever though.** If that's intentional, it's a strong argument for it — Kkullm reads as a Korean-rooted word with an AI/LLM pun hiding in plain sight, which is exactly the kind of thing that lands well with a technical audience who'll get it.

If you lean into that angle, it's a feature not a bug. If you'd rather the name stand purely on its own phonetics, **Kkullim** is still the cleaner choice.

> I picked **Kkullm**.
